The Section filter is used to specify what parts of a Specification page must be interpreted by LivingDoc.
When the specification page contains sections identified by tags, it is possible to execute only those parts of the specification that are relevant. All specified sections are comma separated's.

If no general sections are specified, all of the document will be interpreted :

rule for Section filter
Document content specified sections to filter interpreted elements?
[a table] none [a table]
[a table] unix [a table]

A section ends when another section is found, if not, it ends at the end of the document.

rule for Section filter
Document content specified sections to filter interpreted elements?

[unix]

****
[section]
[mac]
****
[a second table] 		unix [a first table]
[section]
[unix]
****
[a first table]
****
[a second table] 		unix [a first table][a second table]
